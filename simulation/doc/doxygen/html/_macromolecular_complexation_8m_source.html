00003 % @wholeCellModelID Process_MacromolecularComplexation
00004 % @name             Macromolecular complexation
00005 % @description
00006 %   Biology
00007 %   ===============
00008 %   An important step in the synthesis of functional enzymes is the
00009 %   stochiometric formation of macromolecular complexes. Macromolecular
00010 %   complexation is kinetically fast, and energetically favorable.
00011 %   Consequently, the model assumes that complexation is limited only by
00012 %   subunit availability, and proceeds to completion rapidly. In addition,
00013 %   we assume that each complex forms with the same specific rate.
00014 %
00015 %   Knowledge Base
00016 %   ===============
00017 %   As of 8/17/2010 the M. genitalium knowledge base contained 155
00018 %   macromolecular complexes involving 262 protein monomer species and 5 RNA
00019 %   species, and 277 links between protein/RNA/complex subunit species and
00020 %   complexes. Of these 155 complexes, detailed information is available on
00021 %   the formation of 6, and these six complexes are formed in other processes.
00022 %   The remaining 149 are formed by this process. The following table lists
00023 %   several statistics about the macromolecular complexation network.
00024 %
00025 %      Statistic                                      Value
00026 %      ===========================================    ===========
00027 %      Mean no. subunit species per complex           1.8 +/- 3.2
00028 %      Min, Max no. subunits species per complex      1 - 34
00029 %      Mean no. subunits per complex                  5 +/- 15.6
00030 %      Min, Max no. subunits per complex              2 - 192
00031 %      Mean no. complexes per monomer species         1 +/ 0.1
00032 %      Min, Max no. complexes per monomer species     1 - 2
00033 %      No. subunits participating in >1 complexes     5
00034 %      No. subunits which are themselves complexes    5
00035 %      No. complexes formed in this process            149
00036 %      No. complexes formed in other processes          6
00037 %
00038 %   Representation
00039 %   ===============
00040 %   Two properties are used to represent the counts of the 149 macromolecular
00041 %   complexes formed by this process (complexs), and of free macromolecular
00042 %   complex subunits (substrates).
00043 %
00044 %   Four properties are used to represent the structure of the macromolecular
00045 %   complex - subunit network: complexComposition, complexNetworks,
00046 %   substrates2complexNetworks, and complexs2complexNetworks. complexComposition
00047 %   is an adjancency matrix between subunits and complexes populated from the
00048 %   knowledge base; entries contain the number of subunits of each type in each
00049 %   complex. complexNetworks is cell array containing the clustering of
00050 %   complexComposition built by findNonInteractingRowsAndColumns called by
00051 %   initializeConstants; each entry represents a disconnected part of the
00052 %   macromolecular complex - subunit network, which can be simulated separate from
00053 %   all other disconnected parts. substrates2complexNetworks and
00054 %   complexs2complexNetworks represent mappings between substrates and complexs
00055 %   and the disconnected parts of the macromolecular complex - subunit network
00056 %   stored in complexNetworks.
00057 %
00058 %   Initialization
00059 %   ===============
00060 %   Macromolecular complexes are initialized up to the amounts of RNA and
00061 %   protein subunits initialized by other processes.
00062 %
00063 %   Simulation
00064 %   ===============
00065 %   Macromolecular complexes are formed assuming:
00066 %   1) Complexation is highly energetically favorable and
00067 %   2) Complexation is fast, and thus
00068 %   3) Macromolecular complexes are formed to completion; that is until
00069 %      there are insufficient free monomers to form additional complexes.
00070 %
00071 %   Complexes are formed according to Monte Carlo simulation for
00072 %   each independent protein complex network (previously established by
00073 %   initializeConstants and stored in complexNetworks,
00074 %   substrates2complexNetworks, and complexs2complexNetworks). First,
00075 %   we use mass-action kinetics to compute the relative formation rate of
00076 %   each complex. Specifically we compute the relative formation rate as
00077 %   the product of the concentration of all monomers raised to the power of
00078 %   their stoichiometries within the complex. Second we stochastically form
00079 %   complexes according the computed formation rates. This is repeated until
00080 %   no further complexes can form.

00334 function proteinComplexs = buildProteinComplexs_montecarlokinetic(...
00335         totalProteinMonomers, proteinComplexMatrix, randStream)
00336     nComplexs = size(proteinComplexMatrix, 2);
00337     proteinComplexs = zeros(nComplexs, 1);
00338 
00339     %build complexes stochastically
00340     while true
00341         %compute rate of building each complex
00342         cumprob = buildProteinComplexs_rates_collisionTheory(...
00343             totalProteinMonomers, proteinComplexMatrix, 'cumulative probability');
00344 
00345         %stop when we can't make any more complexes
00346         if isnan(cumprob(1)); break; end;
00347 
00348         %select complex
00349         selectedComplex = find(randStream.rand() < cumprob, 1, 'first');
00350         if isempty(selectedComplex)
00351             selectedComplex = find(cumprob == 1, 1, 'first');
00352         end
00353 
00354         %mass balance
00355         proteinComplexs(selectedComplex) = proteinComplexs(selectedComplex) + 1;
00356         totalProteinMonomers = totalProteinMonomers - proteinComplexMatrix(:, selectedComplex);

00360 function rates = buildProteinComplexs_rates_collisionTheory(...
00361         totalProteinMonomers, proteinComplexMatrix, normalization)
00362     %compute rate of building each complex
00363     %Assumes that complexes form by all subunits simultaneously colliding.
00364     %However, protein complexes do not assemble this way. Rather subunits are
00365     %added sequentially. Thus this method will will underestimate the rate of
00366     %formation of larger protein complexes relative to smaller complexes.
00367     %Note: to get correct units subtract one from the sum(coefficients)
00368     %term. Its not necessary to get the units correct because this factor
00369     %is later normalized out.
00370     %TODO: replace mean(totalProteinMonomers) with better estimate of volume
00371     rates = prod((totalProteinMonomers(:, ones(size(proteinComplexMatrix,2), 1)) / mean(totalProteinMonomers)) .^ proteinComplexMatrix, 1)';
00372     
00373     %set rate to zero if upper zero
00374     ub = buildProteinComplexs_bounds(totalProteinMonomers, proteinComplexMatrix);
00375     rates(ub == 0) = 0;
00376     
00377     if nargin >= 3
00378         switch normalization(1)
00379             case 'c' %cumulative probability
00380                 rates = cumsum(rates);
00381                 rates = rates / rates(end);
00382             case 'p' %probability
00383                 rates = rates / sum(rates);
00384             otherwise
00385                 throw(MException('MacromolecularComplexation:error', 'Invalid normalization'));
00386         end
00387     end
00388 end
