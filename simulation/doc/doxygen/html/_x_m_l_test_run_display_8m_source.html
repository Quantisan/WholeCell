00230             %    new_stack = filterStack(stack) removes from the input stack trace calls

00232             %    meaningful to the user.
00233             
00234             % Testing stack traces follow this common pattern:
00235             %
00236             % 1. The first function call in the trace is often one of the assert functions
00237             % in the framework directory.  This is useful to see.
00238             %
00239             % 2. The next function calls are in the user-written test functions/methods and
00240             % the user-written code under test.  These calls are useful to see.
00241             %
00242             % 3. The final set of function calls are methods in the various framework
00243             % classes.  There are usually several of these calls, which clutter up the
00244             % stack display without being that useful.
00245             %
00246             % The pattern above suggests the following stack filtering strategy: Once the
00247             % stack trace has left the framework directory, do not follow the stack trace back
00248             % into the framework directory.
00249

00251             last_keeper = numel(stack);

00254                 directory = fileparts(stack(k).file);
00255                 if have_left_mtest_directory
00256                     if strcmp(directory, mtest_directory)

00258                         last_keeper = k - 1;

00260                     end

00262                     if ~strcmp(directory, mtest_directory)

00265                 end
